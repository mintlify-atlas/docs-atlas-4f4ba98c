Get your local instance of Login Machine up and running in minutes.

## Prerequisites

Before you begin, make sure you have:

- **Node.js 20+** installed on your machine
- An **Anthropic API key** for Claude Sonnet 4.5 ([sign up here](https://console.anthropic.com/))
- A **BrowserBase account** for cloud browser automation ([sign up here](https://www.browserbase.com/))

<Note>
  New to BrowserBase or Anthropic? Check out the [Environment Setup](/environment-setup) guide for detailed instructions on obtaining API keys.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Login Machine repository from GitHub:

    ```bash
    git clone https://github.com/RichardHruby/login-machine.git
    cd login-machine
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required npm packages:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your API keys:

    ```bash
    cp .env.example .env.local
    ```

    Open `.env.local` and add your credentials:

    ```bash .env.local
    # Required: Anthropic API key for Claude
    ANTHROPIC_API_KEY=sk-ant-xxxxx

    # Required: BrowserBase credentials
    BROWSERBASE_API_KEY=bb_xxxxx
    BROWSERBASE_PROJECT_ID=xxxxx

    # Optional: PostHog analytics (leave blank to disable)
    NEXT_PUBLIC_POSTHOG_KEY=
    NEXT_PUBLIC_POSTHOG_HOST=
    ```

    <Warning>
      Never commit your `.env.local` file to version control. It's already included in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Launch the Next.js development server:

    ```bash
    npm run dev
    ```

    The app will start at [http://localhost:3000](http://localhost:3000).
  </Step>

  <Step title="Try your first login">
    1. Open [http://localhost:3000](http://localhost:3000) in your browser
    2. Enter a login URL (e.g., `https://github.com/login`)
    3. Watch as the AI analyzes the page in real-time
    4. Fill in the credentials when prompted
    5. The agent will submit the form and continue until logged in

    <Info>
      The browser preview shows the live BrowserBase session. You can see exactly what the agent sees.
    </Info>
  </Step>
</Steps>

## What Happens Next?

After starting a session, Login Machine will:

1. **Launch a cloud browser** via BrowserBase
2. **Navigate** to your login URL
3. **Take a screenshot** and extract the page HTML
4. **Classify the screen** using Claude Sonnet 4.5
5. **Request input** if credentials are needed
6. **Submit the form** and wait for the next screen
7. **Repeat** until successfully logged in or an error occurs

## Building for Production

To create an optimized production build:

<CodeGroup>
```bash npm
npm run build
npm run start
```

```bash yarn
yarn build
yarn start
```

```bash pnpm
pnpm build

```
</CodeGroup>

The production build includes:
- Server-side rendering (SSR)
- Optimized static assets
- Code splitting and tree shaking
- Environment variable validation

## Common Issues

<AccordionGroup>
  <Accordion title="Error: ANTHROPIC_API_KEY must be set">
    Make sure you've created a `.env.local` file with your Anthropic API key. The key should start with `sk-ant-`.

    ```bash
    cp .env.example .env.local
    # Edit .env.local and add your key
    ```
  </Accordion>

  <Accordion title="Error: BROWSERBASE_API_KEY must be set">
    You need a BrowserBase account to run the browser automation. Sign up at [browserbase.com](https://www.browserbase.com/) and add your API key to `.env.local`.
  </Accordion>

  <Accordion title="Browser session times out">
    BrowserBase free tier includes 10 hours/month. Check your usage at [browserbase.com/dashboard](https://www.browserbase.com/dashboard). Each login session typically uses 2-5 minutes.
  </Accordion>

  <Accordion title="Port 3000 is already in use">
    Kill the existing process or run on a different port:

    ```bash
    npm run dev -- -p 3001
    ```
  </Accordion>

  <Accordion title="Login fails with 'Max retries exceeded'">
    Some pages may take longer to load. The agent retries up to 12 times for loading screens. If it still fails:
    - Check the browser preview for errors
    - Try a simpler login page first
    - See [Troubleshooting](/guides/troubleshooting) for more tips
  </Accordion>
</AccordionGroup>
